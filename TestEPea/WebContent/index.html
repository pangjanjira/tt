eGP "SETUP" Request Management

There are 3 Steps to manage the eGP "SETUP" request

  • 1. Control Account
  • 2. Monitor Request
  • 3. Inquiry Request

Control Account

This step is to manage the ALS accounts which has to be closely monitored. User has to filled in the ALS account, message code, and message description (the reason on why this account has to be monitored).

Monitor Request

The requests which own by the controlled account will be flagged as "pending approve" for user to review before approve or reject the requests.

In this step, user will use the "Monitor Request" screen to monitor the list of the controlled account's request.

After reviewing the request, user can approve or reject the request.

- After approve, the request will be sent to ALS for further processing.
   If ALS reject the request, the request will be flagged as "rejected from ALS".
   The request still be seen in the "Monitor Request" screen, user can review the request again before re-approve or reject.
   If ALS approve the request, the request will be flagged as "approved request" and send response back to eGP (with "Approved" status).

- After reject, the request will be responsed back to eGP (with "Rejected" status).
   The request will be flagged as "rejected request" and and send response back to eGP (with "Rejected" status).

Inquiry Request

The SETUP / CANCEL / CLAIM requests which already responsed back to eGP can be inquired in the "Inquiry Request" screen.
